## 2. 設計思想

### 5つの設計原則

| 原則 | 内容 |
|---|---|
| **① 安定性最優先** | クラッシュしない。NAS接続断もクラッシュではなく「状態」として扱う |
| **② 触らなくても快適** | デフォルト設定で十分快適に読める |
| **③ 触ればもっと自分好みに** | 設定を深く潜れば細かくカスタマイズできる |
| **④ よく使う設定はプリセットで一発** | 複数のプリセットを保存・即時切り替え |
| **⑤ UIは自分好みにカスタマイズできる** | ツールバーの中身・表示タイミングを変更可能 |

### NASファーストの思想

```
NASもローカルも「場所」として統一的に扱う
ファイルは参照するだけ・汚さない・移動させない
メタデータ（評価・タグ等）はアプリ内DBで管理
サムネイルのみキャッシュとして内部に保存
```

### Comic Glassリスペクト

本アプリはiOS用Comic Glassに強くインスパイアされている。
UIの設計思想・操作概念を参考にしており、アプリ説明欄にその旨を明記する。

## 5. 機能仕様

### ストレージ・ファイルアクセス

#### 「場所」の種類

| 種類 | 例 | 特徴 |
|---|---|---|
| NAS（SMB接続） | //192.168.1.10/manga/ | ストリーミング or DL |
| ローカルフォルダ | /sdcard/Comics/ | 直接参照 |
| アプリ専用フォルダ | 内部ストレージ/ComicVeil/ | 転送ファイルの保管場所 |

#### 内部ストレージの使い方

```
保存するもの：
・SQLiteデータベース（評価・カラーラベル・読書位置・ブックマーク・設定）
・サムネイルキャッシュ（ローカル・NAS両対応）
・NASキャッシュ（STRモード時の一時ファイル / ページ単位キャッシュ）
・アプリ設定（SharedPreferences）

保存しないもの：
・コミックファイル本体（参照のみ・DLモード時を除く）
```

#### DL/STRモード

| モード | 保存先 | 特徴 |
|---|---|---|
| STR（ストリーミング・デフォルト） | アプリキャッシュ（一時） | ZIP：1ページ目から即閲覧開始（Progressive Loading）、RAR：全DL後 |
| DL（ダウンロード） | Downloads/ComicVeil/（永続） | ファイルブラウザからも参照可能 |

#### ZIP Progressive Loading（STRモード）
- ZipArchiveInputStreamでNASからストリームを読み込みながらページを逐次書き出し
- 1ページ目が届いた時点でビューワーを起動
- ページはファイルパス（String）として管理しメモリを節約
- RARは従来通り全DL後に表示（将来的にProgressive対応予定）

#### Intent対応（他アプリからの受け取り）
- ファイルマネージャー等から「Comic Veilで開く」でZIP/RARを直接起動
- content URI → キャッシュにコピー、file URI → そのまま参照

### 読書状態管理

#### 状態の定義

| 状態 | 条件 |
|---|---|
| **未読** | 一度も開いていない、または先頭ページから送っていない |
| **読書中** | 1ページ以上送り、最終ページ未到達 |
| **既読** | 最終ページに到達している |

#### 自動遷移ルール

| 操作 | 結果 |
|---|---|
| コミックを開く（先頭ページ） | 状態変化なし（未読のまま） |
| 1ページ以上送る | 読書中 |
| 最終ページに到達 | 既読 |
| 最終ページから1枚以上戻す | 読書中 |

#### 手動変更時のページ位置連動

| 変更 | ページ位置の扱い |
|---|---|
| → 未読 | 1ページ目にリセット |
| → 既読 | 最終ページに設定 |
| → 読書中 | 手動選択不可（自動遷移のみ） |

### サムネイル管理

| 操作 | 対象 | アクセス方法 |
|---|---|---|
| 個別再作成 | 1ファイル | ファイル情報ポップアップ |
| フォルダ内再作成 | 現在のフォルダのみ | Aboutメニュー |
| 全体再作成 | ライブラリ全体 | Settings → キャッシュ管理 |

#### NASサムネイルの取得順序
1. サムネイルキャッシュ（thumbnails/nas_*.jpg）があれば即返却
2. STRキャッシュ（ページキャッシュ）があればそこから生成
3. NASから先頭2MBのみ取得してZIPの先頭画像を抽出
4. RAR・パスワード付きZIPはスキップ（アイコン表示）

## 7. データ設計

### DBテーブル構成（v4現在）

```sql
-- 読書位置（実装済み）
reading_progress
  filePath（PK） / currentPage / totalPages / lastReadAt

-- ファイルのメタデータ（v4・実装済み）
files
  filePath（PK） / status（未読0/読書中1/既読2） / rating（0〜5）
  colorLabel（0=なし/1=赤/2=橙/3=緑/4=青/5=紫） / registeredAt

-- ブックマーク（v3・実装済み）
bookmarks
  id / filePath / page / createdAt

-- タグ（未実装）
tags
  id / 名前

-- ファイルとタグの紐付け（未実装）
file_tags
  file_id / tag_id

-- プリセット（未実装）
presets
  id / 名前（A〜E）/ 設定JSON
```

### DB マイグレーション履歴
```
v1 → v2：files テーブル追加
v2 → v3：bookmarks テーブル追加
v3 → v4：files に rating / colorLabel カラム追加
```

## 8. 対応フォーマット

### 画像フォーマット

| カテゴリ | フォーマット | 優先度 |

| Web主流 | JPEG, PNG, WebP, GIF | 最優先 |
| 次世代 | HEIC, AVIF, JPEG XL | 高 |

### アーカイブフォーマット
